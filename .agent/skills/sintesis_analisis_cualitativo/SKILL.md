# Síntesis y Análisis Cualitativo

Esta habilidad permite transformar la "data cruda" de la investigación (videos, notas, fotos) en información estructurada, conocimiento y, finalmente, criterios de diseño para la innovación.

## Cuándo usar esta habilidad
- Al finalizar una fase de recolección de datos etnográficos o entrevistas.
- Cuando necesites crear Personas o Escenarios basados en investigación real.
- Para traducir hallazgos complejos en requerimientos para ingeniería o diseño.
- Si el usuario solicita un análisis de texto o contenido de documentos de investigación.

## 1. El Proceso de Síntesis (Sensemaking)
Basado en el enfoque de Jon Kolko para manejar la complejidad:

1. **Búsqueda de Instancias:** Identificar momentos específicos, citas o comportamientos en los datos.
2. **Identificación de Patrones:** Agrupar instancias similares para encontrar comportamientos recurrentes.
3. **Externalización:** Sacar los datos de las carpetas/archivos y hacerlos visibles (post-its, diagramas en paredes/pizarras).
4. **Modelado:** Crear una narrativa coherente que explique la relación entre el usuario y el producto.

## 2. Herramientas de Modelado

### Personas y Arquetipos
- **Basado en Evidencia:** No son personajes ficticios ideales, sino representaciones de patrones de comportamiento observados.
- **Enfoque en Metas:** ¿Qué intentan lograr? ¿Qué les impide lograrlo?

### Escenarios de Uso
- Narrativas que sitúan a la Persona en un contexto (Environment) realizando una Actividad (según AEIOU).
- Ayudan a visualizar cómo el nuevo diseño resolverá problemas detectados.

### Análisis de Texto y Contenido
- Identificación de códigos y categorías temáticas en transcripciones de entrevistas.
- Uso de herramientas visuales para ver la frecuencia y relación entre temas.

## 3. Principios de Síntesis
1. **Rigor sobre Objetividad:** El objetivo no es ser estadísticamente "neutro", sino ser riguroso en la interpretación de las particularidades y "outliers" (casos únicos que pueden ser fuente de innovación).
2. **Abducción:** Generar la explicación más probable para un conjunto complejo de observaciones.
3. **Transformación:** Los datos no hablan por sí solos; deben ser transformados activamente por el investigador/diseñador en *insights*.
4. **Visualización:** El uso de diagramas y modelos visuales es esencial para comunicar la complejidad de forma comprensible.

## 4. Estructura de Salida Recomendada (Informe de Síntesis)
- **Hallazgos Clave (Insights):** ¿Qué aprendimos de nuevo? (Ej. "Los usuarios no usan X porque prefieren el workaround Y").
- **Implicancias de Diseño:** ¿Qué debe tener o hacer el producto basándose en el insight?
- **Modelos Visuales:** Diagramas de flujo, Personas, Mapas de Experiencia.
- **Evidencia Directa:** Citas literales o referencias a clips de video específicos.
